# 07 コード解析（Code Analysis）

---

## 概要

既存コードベースの品質・構造・問題点を Claude Code が体系的に解析するためのプロンプトです。

---

## Claude Code 起動コマンド

```
以下のコードベースを解析してください。

【解析対象】
[ファイルパス or ディレクトリ]

【解析観点】
1. コード品質（複雑度・重複・命名規則）
2. アーキテクチャ（依存関係・レイヤ分離・設計パターン）
3. セキュリティ（脆弱なパターン・入力検証・認証）
4. パフォーマンス（N+1・不必要な処理・メモリリーク）
5. テスト（カバレッジ・テストの質・フレークネス）

各観点について問題点と改善提案をMarkdownレポートで出力してください。
```

---

## 解析項目詳細

### 1. コード品質解析

| 項目 | 確認内容 | ツール |
|------|---------|--------|
| 循環複雑度 | 関数の複雑さ（10以下推奨） | SonarQube, ESLint |
| 重複コード | DRY原則違反の検出 | jscpd, SonarQube |
| 命名規則 | 変数・関数・クラス名の一貫性 | ESLint, pylint |
| コメント品質 | 適切なドキュメンテーション | JSDoc, pydocstyle |

### 2. アーキテクチャ解析

```
解析観点:
- レイヤ間の依存方向（上位→下位のみ）
- 循環依存の有無
- 責務の適切な分離（SRP違反）
- インターフェースの適切な定義
```

### 3. セキュリティ解析

```
確認パターン:
- SQLインジェクション
- XSS（クロスサイトスクリプティング）
- 認証・認可の漏れ
- ハードコードされた認証情報
- 安全でない依存ライブラリ
```

## 定期解析の自動化

```yaml
# .github/workflows/code-analysis.yml
name: Code Analysis
on:
  pull_request:
    types: [opened, synchronize]
  schedule:
    - cron: '0 9 * * 1'  # 毎週月曜9時

jobs:
  analyze:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Run Claude Code Analysis
        run: |
          claude "07_コード解析(CodeAnalysis).md のプロンプトでコード解析を実行"
```
