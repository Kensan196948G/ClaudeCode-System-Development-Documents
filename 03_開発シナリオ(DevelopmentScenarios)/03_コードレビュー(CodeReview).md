# 03 コードレビュー（Code Review）

## Claude Code 起動コマンド

```bash
claude --dangerously-skip-permissions
```

## プロンプト指示

```
以下のコードまたはプルリクエストをレビューしてください。

【レビュー対象】
- ファイル / PR: [FILE_PATH or PR_URL or ブランチ名]
- 変更内容の概要: [どんな変更か]

【レビュー観点（Agent Teams: QA / Architect / Security）】
1. 🔍 正確性
   - ロジックのバグ・エッジケースの見落とし
   - 型安全性・null 安全性
2. 🏗 設計・保守性
   - SOLID 原則への準拠
   - 関数・クラスの責務の明確さ
   - 命名の適切さ
3. ⚡ パフォーマンス
   - 不必要なループ・再計算
   - N+1 問題・メモリリーク
4. 🔐 セキュリティ
   - インジェクション・XSS・CSRF リスク
   - 機密情報のハードコード
5. 🧪 テスト
   - テストカバレッジの十分さ
   - テストケースの網羅性

【出力形式】
- 重大度: 🔴 Critical / 🟠 Major / 🟡 Minor / 🟢 Suggestion
- 各指摘に対して改善コードを提示する
- 問題がなければ LGTM と明記する

レビュー完了後、自動修正可能な Minor 以下の指摘は修正してコミットしてください。
```

## 使用場面

- PR マージ前の品質チェック
- チームメンバーのコードのセカンドオピニオン
- 自分のコードの客観的な評価

## ポイント

- `git diff main` の出力を貼り付けると精度が上がる
- 「自動修正可能な指摘は修正してコミット」を入れることで効率化できる
