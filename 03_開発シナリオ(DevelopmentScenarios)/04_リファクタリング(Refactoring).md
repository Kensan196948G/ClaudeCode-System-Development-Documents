# 04 リファクタリング（Refactoring）

## Claude Code 起動コマンド

```bash
claude --dangerously-skip-permissions
```

## プロンプト指示

```
以下のコードをリファクタリングしてください。

【対象】
- ファイル / モジュール: [FILE_PATH or MODULE_NAME]
- 現在の問題点:
  - [ ] コードが長すぎる（関数が100行超など）
  - [ ] 重複コードが多い（DRY 原則違反）
  - [ ] 命名が不明確
  - [ ] テストが書きにくい構造
  - [ ] 型が不十分
  - [ ] その他: [詳細]

【方針（Architect / 該当 Dev で議論）】
- 外部インターフェース（API・関数シグネチャ）は変更しない
- 既存テストをすべてパスさせる
- リファクタリング後に新規テストを追加する
- パフォーマンスを悪化させない

【実行してほしいこと】
1. 現状コードを分析して問題点を整理する
2. リファクタリング計画を提示して承認を得る
3. 段階的にリファクタリングを実施する
4. 各ステップでテストを実行して回帰がないことを確認する
5. 変更の意図をコメントに記載する
6. git commit（refactor: [対象モジュール]）する

【完了基準】
- 全テストがパス
- lint / typecheck がパス
- 循環的複雑度が改善されている
```

## 使用場面

- 技術的負債の解消
- 大規模機能追加の前準備
- オンボーディング前の可読性向上

## ポイント

- 一度に大きく変えず、段階的リファクタリングを指示するとリスクが下がる
