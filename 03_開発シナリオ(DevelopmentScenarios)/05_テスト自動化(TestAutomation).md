# 05 テスト自動化（Test Automation）

## Claude Code 起動コマンド

```bash
claude --dangerously-skip-permissions
```

## プロンプト指示

```
以下のモジュール / 機能に対してテストを自動生成・整備してください。

【対象】
- テスト対象: [FILE_PATH or MODULE_NAME]
- テストフレームワーク: [Jest / Vitest / pytest / Go test など]
- 現在のカバレッジ: [xx% または「未計測」]

【テスト種別（Tester Agent が担当）】
- [ ] ユニットテスト（純粋関数・クラスメソッド）
- [ ] 統合テスト（API エンドポイント・DB 操作）
- [ ] E2E テスト（Playwright / Cypress）
- [ ] スナップショットテスト（UI コンポーネント）

【実行してほしいこと】
1. 対象コードを解析してテスト戦略を提示する
2. 境界値・異常系・ハッピーパスをすべてカバーするテストケースを設計する
3. テストコードを生成する
4. テストを実行してすべてパスすることを確認する
5. カバレッジレポートを生成して改善前後を比較する
6. カバレッジが 80% 未満の場合は追加テストを生成する
7. git commit（test: [対象モジュール] のテストを追加）する

【品質基準】
- ライン / ブランチカバレッジ 80% 以上
- テストは独立して実行できる（副作用なし）
- モックは最小限にする
```

## 使用場面

- CI に自動テストを追加したいとき
- リファクタリング前の安全網を作るとき
- 新機能のテストを効率的に書きたいとき

## ポイント

- カバレッジ目標を明示することで必要十分なテストが生成される
