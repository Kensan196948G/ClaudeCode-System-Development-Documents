# 06 機能追加（Feature Addition）

---

## 概要

既存プロジェクトへの新機能追加を Claude Code が自律的に実装するためのプロンプトです。

---

## Claude Code 起動コマンド

```
以下の機能追加タスクを実行してください。

【追加する機能】
[機能名・概要を記述]

【要件】
- [要件1]
- [要件2]
- [要件3]

【受け入れ基準】
- [ ] [確認項目1]
- [ ] [確認項目2]
- [ ] [確認項目3]

【制約事項】
- 既存APIの後方互換性を維持すること
- テストカバレッジを維持すること（現状以上）
- コーディング規約に準拠すること

Triple Loop アーキテクチャで実装してください。
```

---

## 実行フロー

```
1. Monitor Loop
   ├── 既存コードベースを解析
   ├── 追加箇所・影響範囲を特定
   └── 実装計画を策定

2. Build Loop
   ├── インターフェース定義
   ├── 実装コード作成
   ├── ユニットテスト作成
   ├── 結合テスト作成
   └── Gitコミット

3. Verify Loop
   ├── コードレビュー
   ├── テスト全通過確認
   ├── 既存機能への影響確認
   └── ドキュメント更新
```

---

## チェックリスト

### 実装前
- [ ] 要件・スコープが明確に定義されている
- [ ] 既存コードの構造を把握している
- [ ] テスト戦略が決まっている

### 実装中
- [ ] 既存の設計パターンに従っている
- [ ] エラーハンドリングを実装している
- [ ] ログ出力を適切に追加している

### 実装後
- [ ] ユニットテストが通過している
- [ ] 結合テストが通過している
- [ ] 既存テストがすべて通過している
- [ ] コードレビューが完了している
- [ ] ドキュメントが更新されている

---

## プロンプトカスタマイズ例

### REST API エンドポイント追加

```
POST /api/v1/users エンドポイントを追加してください。

リクエスト: { name: string, email: string, role: "admin" | "user" }
レスポンス: { id: string, name: string, email: string, role: string, createdAt: string }

バリデーション・認証チェック・エラーレスポンスを含めてください。
対応するユニットテストと結合テストも作成してください。
```

### フロントエンドコンポーネント追加

```
ユーザー一覧を表示する UserList コンポーネントを作成してください。

- ページネーション対応（1ページ20件）
- 検索フィルター（名前・メール）
- ソート機能（名前・作成日）
- ローディング・エラー状態の表示

Storybook のストーリーとユニットテストも作成してください。
```
