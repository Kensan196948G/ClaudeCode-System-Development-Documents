# 14 依存関係更新（Dependency Update）

## Claude Code 起動コマンド

```bash
claude --dangerously-skip-permissions
```

## プロンプト指示

```
このプロジェクトの依存関係を安全に最新化してください。

【対象】
- パッケージマネージャー: [npm / yarn / pnpm / pip / cargo / go]
- 更新スコープ:
  - [ ] セキュリティパッチのみ（安全）
  - [ ] マイナーバージョンアップ（比較的安全）
  - [ ] メジャーバージョンアップ（破壊的変更の可能性あり）

【実行してほしいこと（Ops / Security Agent が担当）】
1. 現在の依存関係を一覧化し、更新可能なパッケージを確認する
2. セキュリティ脆弱性のある依存関係を優先して修正する
3. 更新によって破壊的変更がないかを各パッケージの CHANGELOG で確認する
4. 段階的に更新を適用する（一度に全部変えない）
5. 各更新後にテストを実行して回帰がないことを確認する
6. メジャーバージョンアップが必要な場合は影響範囲を調査してコードを修正する
7. package-lock.json / requirements.txt をコミットする
8. git commit（chore: 依存関係を更新）する

【優先順位】
1. 🔴 CVE が報告されているパッケージ
2. 🟠 メンテナンス終了 (EOL) のパッケージ
3. 🟡 マイナーアップデートのパッケージ
4. 🟢 メジャーアップデートのパッケージ
```

## 使用場面

- 定期的な依存関係メンテナンス
- Dependabot アラートへの対応
- Node.js / Python のバージョンアップ

## ポイント

- 「一度に全部更新しない」という制約が回帰リスクを劇的に下げる
