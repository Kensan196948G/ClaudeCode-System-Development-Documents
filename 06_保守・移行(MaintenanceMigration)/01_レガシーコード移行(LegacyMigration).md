# 13 レガシーコード移行（Legacy Code Migration）

## Claude Code 起動コマンド

```bash
claude --dangerously-skip-permissions
```

## プロンプト指示

```
以下のレガシーコードを現代的な技術スタックに移行してください。

【現状】
- 現在の技術: [JavaScript / Python 2 / jQuery / PHP5 など]
- 移行先の技術: [TypeScript / Python 3.12 / React / PHP 8 など]
- 対象ファイル / モジュール: [PATH or 全体]
- コードベースの規模: [ファイル数・行数の概算]

【移行方針（Architect / CTO Agent が設計）】
- 段階的移行（Strangler Fig パターン）を採用する
- 既存機能を完全に維持する
- テストを先に書いてから移行する（テストファースト移行）
- パフォーマンスを悪化させない

【実行してほしいこと】
1. コードベースを解析して移行の難易度・リスクを評価する
2. 移行計画（フェーズ分け・優先度）を提示して承認を得る
3. フェーズ1: テストを作成する（移行前の動作を保証）
4. フェーズ2: 段階的に新技術へ移行する
5. フェーズ3: 旧コードを削除する
6. 各フェーズでテストを実行して回帰がないことを確認する
7. 移行完了後に型安全性・lint をチェックする
8. git commit（migrate: [対象] を [新技術] に移行）する

【リスク管理】
- 移行中は旧コードと新コードを並行して動かせる設計にする
- ロールバック手順を事前に準備する
```

## 使用場面

- JavaScript → TypeScript 移行
- CommonJS → ESModules 移行
- レガシー API の REST 化・GraphQL 化

## ポイント

- 段階的移行を強調することで一気に変えるリスクを避けられる
