# 15 インシデント対応（Incident Response）

## Claude Code 起動コマンド

```bash
claude --dangerously-skip-permissions
```

## プロンプト指示

```
本番環境でインシデントが発生しています。緊急対応してください。

【インシデント概要】
- 発生時刻: [YYYY-MM-DD HH:MM JST]
- 影響範囲: [全ユーザー / 一部ユーザー / 特定機能]
- 症状:
  - [ ] サービス全停止（500 エラー多発）
  - [ ] 特定機能が動作しない
  - [ ] パフォーマンス劣化（レスポンスタイム xx ms → xx ms）
  - [ ] データ不整合
- エラーログ:
  ```
  [エラーログをここに貼り付け]
  ```

【緊急対応手順（CTO / Ops / 該当 Dev Agent が担当）】
1. 【即時】ログ・メトリクスを確認して根本原因を特定する
2. 【即時】影響を最小化する暫定対処（フィーチャーフラグ OFF / 旧バージョンへのロールバック）
3. 【短期】根本修正を実装してテストする
4. 【確認後】本番環境へ適用する（git push はユーザー確認後）
5. 【事後】インシデントレポートを生成する（docs/incidents/YYYY-MM-DD.md）

【エスカレーション基準】
- 30 分で解決しない場合 → エスカレーション候補者に通知
- データロスの可能性がある場合 → 即時エスカレーション

完了後にポストモーテムドキュメントを作成してください。
```

## 使用場面

- 本番障害の緊急対応
- パフォーマンス劣化の原因調査
- データ不整合の修正

## ポイント

- git push（本番適用）は必ずユーザーが確認してから行う
- ポストモーテムを残すことで再発防止に繋げる
