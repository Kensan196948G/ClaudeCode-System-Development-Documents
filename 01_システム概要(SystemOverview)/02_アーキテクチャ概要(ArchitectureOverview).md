# 02 アーキテクチャ概要（Architecture Overview）

---

## システム全体像

ClaudeCode 自律型開発システムは **Triple Loop アーキテクチャ** を中核とし、  
AI エージェントが Monitor → Build → Verify の3ループを自律的に繰り返すことで、  
高品質なソフトウェア開発を継続的に実現します。

```
┌─────────────────────────────────────────────────────────┐
│              ClaudeCode 自律開発システム                   │
│                                                         │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐  │
│  │ Monitor Loop │→ │  Build Loop  │→ │ Verify Loop  │  │
│  │  (監視・計画) │  │  (実装・構築) │  │  (検証・確認) │  │
│  └──────────────┘  └──────────────┘  └──────────────┘  │
│         ↑                                      │        │
│         └──────────────────────────────────────┘        │
│                    自律ループ継続                          │
└─────────────────────────────────────────────────────────┘
```

---

## Triple Loop アーキテクチャ詳細

### 1. Monitor Loop（監視・計画ループ）

**目的**: 現状把握・タスク優先順位決定・次アクション計画

| 項目 | 内容 |
|------|------|
| 入力 | リポジトリ状態、エラーログ、ユーザー要求 |
| 処理 | コード解析、課題検出、優先度付け |
| 出力 | タスクリスト、実行計画、リスク評価 |
| 周期 | 15分ごと / トリガー発生時 |

### 2. Build Loop（実装・構築ループ）

**目的**: 計画に基づくコード実装・テスト・コミット

| 項目 | 内容 |
|------|------|
| 入力 | Monitor Loop の実行計画 |
| 処理 | コード生成、テスト実行、デバッグ |
| 出力 | 動作するコード、Gitコミット |
| 周期 | タスク単位 |

### 3. Verify Loop（検証・確認ループ）

**目的**: 品質ゲート通過確認・レビュー・フィードバック

| 項目 | 内容 |
|------|------|
| 入力 | Build Loop の成果物 |
| 処理 | コードレビュー、セキュリティ検査、パフォーマンス確認 |
| 出力 | 合格 / 差し戻し判定、改善指示 |
| 周期 | Build Loop 完了後 |

## 7体エージェント構成

```
┌─────────────────────────────────────────────────────────────┐
│                    エージェントチーム                          │
│                                                             │
│  Agent 1: 要件定義      Agent 2: アーキテクト                 │
│  Agent 3: 実装          Agent 4: テスト・品質保証             │
│  Agent 5: セキュリティ  Agent 6: インフラ・DevOps             │
│  Agent 7: ドキュメント・ナレッジ                               │
└─────────────────────────────────────────────────────────────┘
```

各エージェントの詳細は `03_エージェント構成(AgentConfiguration).md` を参照してください。
