# 05 チェックポイント機能（Checkpoints）

---

## 概要

Claude Code のチェックポイント機能は、ツール呼び出しの前に**自動的にプロジェクト状態のスナップショット**を保存します。
失敗した変更や不要な修正を素早く元に戻すことができ、安心して大胆な実装を進める土台となります。

---

## 仕組み

```
[Claude がツール実行前]
      ↓
 スナップショット自動保存
      ↓
[ツール実行（ファイル編集・コマンド実行）]
      ↓
 結果が良ければそのまま続行
 失敗・不要なら /rewind で復元
```

チェックポイントは**ツール呼び出しごと**に作成されます。1回のセッションで複数のポイントに戻ることも可能です。

---

## 復元方法

### 方法 1: `/rewind` コマンド

```
/rewind
```

入力後に **対話型メニュー** が表示され、以下の2種類の復元を選択できます：

| 復元タイプ | 内容 |
|-----------|------|
| **コードのみ復元** | ファイル変更を巻き戻す（会話履歴は保持） |
| **コード＋会話を復元** | ファイル変更と会話コンテキストの両方を巻き戻す |

### 方法 2: Esc × 2（キーボードショートカット）

`Esc` キーを**2回連続**で押すと即座にチェックポイントメニューが表示されます。

```
Esc → Esc
 ↓
チェックポイント一覧が表示
```

---

## 復元の選択肢

```
? チェックポイントを選択してください

  ● 12:34:01 — bash ツール実行前（npm install）
  ● 12:30:15 — Write ツール実行前（src/index.ts）
  ● 12:25:44 — Write ツール実行前（package.json）
  ● 12:20:00 — セッション開始
```

- `↑` / `↓` キーで選択
- `Enter` で確定
- `Esc` でキャンセル

---

## 使用シナリオ

### シナリオ 1: 間違ったファイルを変更してしまった

```
# Claude が誤って本番設定ファイルを更新した場合
/rewind
→ 「コードのみ復元」を選択
→ 変更前の状態に戻る
```

### シナリオ 2: 実験的な実装を試してみたい

```
# あえてリスクを取って実装させる
claude "React 18 の Concurrent Mode に全面移行してみて"

# うまくいかなければ
Esc + Esc → 「コード＋会話を復元」
→ 完全に元の状態から再スタート
```

### シナリオ 3: 途中まで進んだ作業の一部だけ戻す

```
# 複数のチェックポイントから特定のポイントを選択
/rewind
→ 一覧から「3ステップ前のファイル編集前」を選択
→ その時点のコードに戻して別のアプローチを試す
```

---

## 注意事項

| 項目 | 内容 |
|------|------|
| **保存対象** | ファイル変更（Write/Edit ツール実行前）とコマンド実行（bash 実行前） |
| **保存場所** | ローカルのセッション状態（`.claude/checkpoints/`）に一時保存 |
| **セッション終了後** | セッションを閉じるとチェックポイントは削除される |
| **Git との関係** | チェックポイントは Git コミットではない。復元してもGit履歴は変わらない |
| **大規模変更** | 100ファイル超の変更は復元に時間がかかる場合がある |

---

## Triple Loop との統合

Triple Loop 自律開発でチェックポイントを活用する推奨パターン:

```markdown
# CLAUDE.md への追加設定例

## チェックポイント活用ルール
- 各ループ開始前に /rewind ポイントを確認
- Build Loop の大規模変更前にユーザーに確認を要求
- Verify Loop でテスト失敗が続いた場合は /rewind を提案
```
