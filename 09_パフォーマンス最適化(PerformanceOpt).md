# 09 パフォーマンス最適化（Performance Optimization）

## Claude Code 起動コマンド

```bash
claude --dangerously-skip-permissions
```

## プロンプト指示

```
以下の機能・エンドポイントのパフォーマンスを最適化してください。

【現状の課題】
- 対象: [FILE_PATH / API エンドポイント / ページ]
- 現在のパフォーマンス:
  - レスポンスタイム: [xx ms]
  - スループット: [xx req/s]
  - バンドルサイズ: [xx KB]（フロントエンドの場合）
- 問題の症状: [遅い / メモリ使用量が多い / CPU 負荷が高い]

【実行してほしいこと（Architect / DevAPI Agent が担当）】
1. プロファイリングツールを実行して ボトルネックを特定する
2. 特定した問題を優先度順に整理する
3. 最適化計画を提示して承認を得る
4. 以下の観点で最適化を実施する:
   - N+1 クエリの解消（DB クエリの最適化）
   - キャッシュ（Redis / メモリキャッシュ / HTTP キャッシュ）の導入
   - 非同期処理・並列処理の活用
   - バンドルサイズ削減（コード分割・Tree shaking）
   - 画像・アセットの最適化
5. ビフォー / アフターのベンチマーク結果を比較する
6. git commit（perf: [対象] のパフォーマンスを最適化）する

【目標値】
- レスポンスタイム: [目標 ms]
- バンドルサイズ: [目標 KB]
```

## 使用場面

- ページロードが遅い問題の解消
- API レスポンスタイムの改善
- DB クエリの最適化

## ポイント

- 「計測 → 分析 → 最適化 → 再計測」のサイクルを自動化する
