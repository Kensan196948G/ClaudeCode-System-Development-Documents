# /review コマンド

このコマンドを実行すると、現在の変更に対して包括的なコードレビューを行います。

## 実行手順

1. `git diff HEAD` でステージング済みおよび未ステージの変更を取得
2. 変更ファイルを一覧表示
3. 各ファイルについて以下を評価:
   - セキュリティ上の問題（インジェクション・認証・シークレット漏洩）
   - パフォーマンス上の問題（N+1クエリ・不要な再計算）
   - コーディング規約違反（命名・コメント・型定義）
   - テスト不足（新規コードにテストが存在するか）
4. 問題点を重要度（🔴 Critical / 🟡 Warning / 🔵 Info）で分類して出力

## 出力形式

```markdown
## コードレビュー結果

### 📁 変更ファイル一覧
- src/api/user.ts
- tests/api/user.test.ts

---

### src/api/user.ts

#### ✅ 良い点
- 適切な型定義
- エラーハンドリングが実装されている

#### 🔴 Critical（必須修正）
- [行23] SQLインジェクションの危険性: パラメータを直接クエリに埋め込んでいます

#### 🟡 Warning（推奨修正）
- [行45] 未使用の変数 `temp` が残っています

#### 🔵 Info（任意）
- JSDoc コメントを追加するとコードの可読性が向上します

---

### 総合評価
- Critical: 1件 → マージ前に修正が必要です
- Warning: 1件 → 推奨修正
- Info: 1件 → 任意対応
```
