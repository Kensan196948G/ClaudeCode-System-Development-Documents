---

## Triple Loop 動作指示

### 🔍 Monitor Loop（30分）
1. `git status` と `git log --oneline -20` でリポジトリ状態を把握
2. テスト失敗・Lintエラー・TODOコメントを収集
3. 優先度付きタスクリストを `TASKS.md` に更新

### 🔨 Build Loop（2〜4時間）
1. 優先度最上位のタスクから実装を開始
2. コミットは Conventional Commits 規約に従う（`feat:`, `fix:`, `docs:`, `refactor:`, `test:`）
3. 実装後は必ずテストを実行（`npm test` または `pytest`）
4. CIが通らない場合は最大15回まで自動修復を試みる

### 🧪 Verify Loop（3〜4時間）
1. 全テストの通過確認
2. セキュリティ・パフォーマンス上の問題がないか確認
3. コードカバレッジが基準値を満たしているか確認
4. 品質基準を満たした場合のみPR/マージを実行

### コミットメッセージ規約
```

fix: バグ修正
docs: ドキュメント変更
refactor: コードのリファクタリング
test: テストの追加・修正
chore: ビルド・依存関係変更
ci: CI/CD設定変更
```

## 禁止事項

- `.env` ファイルや認証情報をコードにコミットしない
- `main` ブランチへの直接プッシュは禁止（PRを経由すること）
- `console.log` のデバッグコードを本番コードに残さない
- テストなしの機能実装は行わない

## エージェントチーム構成

このプロジェクトでは以下の7体エージェント構成を使用します：

| エージェント | 起動キーワード | 担当 |
|------------|-------------|------|
| 要件定義 | `requirements-agentとして` | Issue・仕様書から要件整理 |
| アーキテクト | `architect-agentとして` | 設計・モジュール分割 |
| 実装 | `implementation-agentとして` | コード実装 |
| テスト | `testing-agentとして` | テスト設計・作成 |
| セキュリティ | `security-agentとして` | 脆弱性チェック |
| DevOps | `devops-agentとして` | CI/CD・インフラ |
| ドキュメント | `docs-agentとして` | 文書作成 |
